# FRs (Requerimientos Funcionales)
Es un documento esencial en el ciclo de vida de desarrollo de software o sistemas. Su objetivo principal es definir de manera detallada y precisa lo que el sistema debe hacer para cumplir con las necesidades y expectativas de los usuarios y las partes interesadas. El propósito de este documento es proporcionar una descripción clara y completa de los requerimientos funcionales necesarios para el desarrollo del sistema. Sirve como guía para los diseñadores, desarrolladores, testers y cualquier otra persona involucrada en el proyecto.
